# Reproducibility map

Every headline finding in `paper/main.tex`, mapped to the committed data file and the committed
script that produces it. Verified by a full re-audit (2026-06-19): the fast seeded scripts were
re-run and reproduced their committed outputs byte-for-byte; the slow L2/backtest scripts read
committed inputs and their committed outputs match the paper. "Re-run" = regenerated in the audit
and confirmed identical. "Committed output matches" = script + input present, output on disk equals
the paper (not re-run live because it is slow).

## Method definitions

Every method/metric in the body is defined in Appendix A (`\section{Methods background}`), now
cross-referenced from the body at first use. The appendix formulas (Kalman recursion, ADF,
Engle-Granger, Newey-West HAC, deflated Sharpe, Ornstein-Uhlenbeck, walk-forward, the four placebos,
the microstructure measures, block bootstrap) were verified against the implementing code and are
correct. The block-bootstrap subsection describes the circular variant the code implements.

## Re-run from scratch

- Figures/tables from committed CSVs/logs: `python scratch/make_figures_v2.py`.
- Kalman controls: `python scratch/kalman_positive_control.py` (seeded, ~1-2 min).
- Reversion persistence: `python scratch/persistence_test.py` (then `persistence_robust.py`).

- Venue/HAC Sharpe: `python scratch/independent_verify.py`, `scratch/cross_exchange.py`.

## Known provenance notes (honest disclosure)

A few descriptive numbers are recorded in the committed writeup `docs/L2_FINDINGS.md` but are not
emitted by an isolated, committed script log. They are internally consistent and were not
contradicted by any data check, but a reproducer should regenerate them from the no-stop backtest
rather than treat the writeup as the primary source:

- median hold ~35 days and "78% of positions do not converge within a quarter" (position-level
  stats of the no-stop reversion book);
- "about a third of the return is a generic survivor comovement floor";
- the specific list of four independent-reimplementation Sharpes (1.95 / 2.15 / 2.16 / 2.18): only
  the 2.18 value has a committed log (`independent_verify.log`); the other three reimplementation
  scripts (`indep3_backtest.py`, `indep_run2.py`, `my_independent_run1.py`) reproduce on running but
  their logs are not committed.
- the "17 to 23% below the flat 5 bps" L2 cost saving is documented via `src/l2_costs.py` in
  `docs/L2_FINDINGS.md` but not in a standalone re-runnable log.
